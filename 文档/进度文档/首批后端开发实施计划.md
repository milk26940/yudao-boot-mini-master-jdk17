# 首批后端开发实施计划

> **范围说明：** 首批后端开发以“商品分类模块”作为最小完整闭环，完成后再进入商品模块、图片模块和校园用户模块。

## 一、当前实施目标

首批目标为完成商品分类后台 CRUD，包含以下内容：

1. 分类数据对象
2. 分类 Mapper
3. 分类 Service
4. 分类 Controller
5. 分类请求与响应 VO
6. 分类单元测试

## 二、实施顺序

### 第 1 步

补充 `yudao-module-system` 的测试依赖，确保后续可执行单元测试。

### 第 2 步

先写分类 Service 单元测试，验证以下行为：

1. 创建分类时返回新建主键
2. 更新不存在分类时抛出业务异常

### 第 3 步

补齐分类模块代码：

1. `SecondCategoryDO`
2. `SecondCategoryMapper`
3. `SecondCategoryService`
4. `SecondCategoryServiceImpl`
5. 分类相关 VO
6. `SecondCategoryController`

### 第 4 步

执行测试与编译验证。

## 三、后续衔接

分类模块完成后，立即进入以下模块：

1. `second_goods`
2. `second_goods_image`
3. `campus_user`
