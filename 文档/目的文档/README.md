# 目的文档目录说明

## 目录用途

本目录用于存放“校园二手物品交易管理系统”项目的目标类、设计类和规划类文档。

为了保证后续文档层次清晰，建议按下面的分类维护。

## 文档分类建议

### 1. 总体类文档

用于说明项目整体定位、目标、范围和总体设计。

建议文件：

1. `校园二手物品交易管理系统-设计方案.md`
2. `项目概述与研究意义.md`
3. `系统总体设计说明.md`

### 2. 需求类文档

用于整理系统面向用户和管理员的功能需求、业务流程和用例。

建议文件：

1. `系统需求分析.md`
2. `功能需求清单.md`
3. `业务流程说明.md`

### 3. 数据类文档

用于描述数据库、表结构、字段设计、E-R 图等内容。

建议文件：

1. `数据库设计.md`
2. `数据表详细设计.md`
3. `E-R图说明.md`

### 4. 实现类文档

用于描述模块实现方案、接口设计、页面设计和技术实现思路。

建议文件：

1. `功能模块设计.md`
2. `接口设计说明.md`
3. `前端页面设计.md`

### 5. 计划类文档

用于描述开发顺序、任务拆分、时间安排和阶段目标。

建议文件：

1. `开发计划.md`
2. `阶段任务拆分.md`
3. `毕业设计进度安排.md`

### 6. 论文类文档

用于辅助毕业论文撰写，整理章节结构和可直接引用的内容。

建议文件：

1. `论文提纲.md`
2. `论文章节素材.md`
3. `系统测试说明.md`

## 推荐命名规范

建议统一采用以下命名方式：

1. 文档名尽量体现主题，例如“数据库设计”“开发计划”“接口设计说明”。
2. 同一类文档尽量避免内容交叉。
3. 总体类文档放前面，细化类文档后续单独拆分。

## 推荐维护顺序

建议按以下顺序继续补全文档：

1. 设计方案
2. 需求分析
3. 数据库设计
4. 功能模块设计
5. 开发计划
6. 论文提纲
