# 校园二手物品交易管理系统设计方案

## 文档定位

本文档用于描述校园二手物品交易管理系统的整体设计方案，内容偏向毕业设计前期的项目定义、功能划分、数据库设计建议和开发实施建议。

建议将本文档作为“总设计文档”使用，后续再拆分出数据库设计、接口设计、页面设计和开发计划等独立文档。

---

## 一、项目概述

### 1.1 项目目标

本项目面向校园场景，设计并实现一个校园二手物品交易管理系统。系统主要服务于在校学生，提供学生注册审核、二手物品发布、物品浏览搜索、求购信息发布、评论交流、交易管理、公告通知和后台管理等功能。

本系统的建设目标如下：

1. 为在校学生提供一个安全、便捷的校内二手交易平台。
2. 通过学生证上传与管理员审核机制，保证平台注册用户具有校园身份。
3. 实现二手物品信息的发布、浏览、检索、管理和交易。
4. 提供求购、评论、公告等辅助功能，增强系统实用性与交互性。
5. 为管理员提供统一的后台管理入口，实现对用户、商品、订单、评论和公告的集中管理。

### 1.2 项目定位

本项目定位为课程设计/毕业设计级别系统，强调以下特点：

1. 功能链条完整，便于论文撰写与答辩展示。
2. 技术实现难度适中，适合在有限周期内完成。
3. 核心业务突出，避免接入高复杂度的真实支付与物流系统。
4. 在保证系统完整性的前提下，对复杂功能采用轻量化实现方式。

### 1.3 系统总体思路

本系统基于现有 `yudao-boot-mini` 工程进行扩展，将现有平台能力作为底座，新增校园二手交易业务模块。

总体实现思路如下：

1. 复用现有系统中的登录认证、验证码、文件上传、公告、邮件通知等基础能力。
2. 新建校园二手交易业务模块，实现商品、求购、评论、订单等领域功能。
3. 保留后台管理端，用于管理员审核和管理系统数据。
4. 增加面向普通学生用户的前台页面，用于注册、登录、浏览、发布和交易。

## 二、需求与角色设计

### 2.1 系统角色设计

系统主要包含两类角色：

#### 2.1.1 普通用户

普通用户即在校学生，拥有以下功能：

1. 注册账号并提交学生证进行审核。
2. 登录系统。
3. 浏览商品和求购信息。
4. 发布、修改、删除自己的二手物品信息。
5. 发布、修改、删除自己的求购信息。
6. 查看商品详情并进行评论。
7. 对感兴趣的商品下单交易。
8. 管理个人资料与个人订单。

#### 2.1.2 系统管理员

系统管理员拥有以下功能：

1. 后台登录。
2. 审核用户注册申请。
3. 管理普通用户信息。
4. 管理商品信息。
5. 管理求购信息。
6. 管理评论信息。
7. 管理订单信息。
8. 发布和管理公告通知。

### 2.2 系统建设范围

本系统面向课程设计/毕业设计场景，范围控制如下：

1. 重点实现完整的业务闭环，而不是追求复杂商业化功能。
2. 优先覆盖注册审核、商品发布、求购、评论、订单、公告和后台管理。
3. 交易采用简化流程，不接入真实支付接口。
4. 推荐功能采用规则推荐，不使用复杂算法模型。

## 三、功能模块设计

### 3.1 功能模块总览

系统功能模块分为以下九类：

1. 用户与认证模块
2. 注册审核模块
3. 二手物品管理模块
4. 求购信息模块
5. 交易订单模块
6. 评论模块
7. 推荐模块
8. 公告通知模块
9. 后台管理模块

### 3.2 各功能模块设计

根据需求分析，系统可划分为以下模块。

#### 3.2.1 用户与认证模块

主要功能：

1. 用户注册。
2. 用户登录。
3. 验证码校验。
4. 个人资料管理。
5. 密码修改。

#### 3.2.2 注册审核模块

主要功能：

1. 用户注册时填写姓名、学号、邮箱等基本信息。
2. 用户上传学生证照片。
3. 管理员审核注册申请。
4. 系统通过邮件通知审核结果。
5. 审核通过后用户方可登录系统。

#### 3.2.3 二手物品管理模块

主要功能：

1. 发布物品信息。
2. 上传物品图片。
3. 设置价格、新旧程度和联系方式。
4. 修改和删除本人发布的物品。
5. 浏览物品列表。
6. 查看物品详情。
7. 按分类、名称、品牌、价格等条件搜索物品。

#### 3.2.4 求购信息模块

主要功能：

1. 用户发布求购信息。
2. 填写物品名称、品牌、型号、预期价格等信息。
3. 查看求购信息列表与详情。
4. 修改和删除本人发布的求购信息。

#### 3.2.5 交易订单模块

本模块采用轻量化设计，不接入真实在线支付。

主要功能：

1. 买家在线下单。
2. 卖家确认订单。
3. 双方线下交易或送货到宿舍。
4. 买家确认收货。
5. 订单状态流转与订单记录查询。

#### 3.2.6 评论模块

主要功能：

1. 登录用户对商品进行评论。
2. 用户回复评论。
3. 展示商品评论列表。
4. 管理员对违规评论进行管理。

#### 3.2.7 推荐模块

本模块采用规则推荐而不是复杂算法推荐。

主要功能：

1. 为登录用户推荐同分类热门商品。
2. 推荐最近发布的商品。
3. 推荐与用户浏览偏好接近的商品。

#### 3.2.8 公告通知模块

主要功能：

1. 管理员发布公告。
2. 用户在首页查看公告内容。
3. 用于展示系统通知、维护信息和活动提示。

#### 3.2.9 后台管理模块

主要功能：

1. 用户管理。
2. 注册审核。
3. 商品管理。
4. 求购管理。
5. 评论管理。
6. 订单管理。
7. 公告管理。
8. 分类管理。

### 3.3 系统核心业务流程

#### 3.3.1 用户注册审核流程

1. 用户填写注册信息。
2. 上传学生证照片。
3. 系统保存注册申请，状态为“待审核”。
4. 管理员在后台审核注册信息。
5. 审核通过后用户状态变为“已通过”，允许登录。
6. 审核不通过则记录驳回原因，并通过邮件通知用户。

#### 3.3.2 物品发布流程

1. 用户登录系统。
2. 进入物品发布页面。
3. 填写商品名称、品牌、价格、新旧程度、描述和联系方式。
4. 上传商品图片。
5. 提交后系统保存商品信息并展示在前台列表中。

#### 3.3.3 商品交易流程

1. 买家浏览商品并查看详情。
2. 买家提交订单。
3. 卖家确认订单。
4. 双方通过线下方式完成交易。
5. 买家确认收货，订单状态更新为完成。

#### 3.3.4 评论交流流程

1. 用户登录后进入商品详情页面。
2. 用户对商品发表评论。
3. 其他用户或卖家可回复评论。
4. 系统以列表或树状形式展示评论信息。

### 4.3 订单状态设计

为降低实现复杂度，订单状态建议采用以下枚举：

1. `0`：待确认
2. `1`：已确认待交易
3. `2`：已完成
4. `3`：已取消

该状态设计满足毕业设计展示需要，也方便进行流程图和状态图描述。

### 4.4 推荐功能设计

推荐功能采用规则推荐方式实现，建议规则如下：

1. 优先推荐同分类商品。
2. 优先推荐浏览量较高的热门商品。
3. 优先推荐最新发布商品。
4. 若系统记录用户浏览行为，可优先推荐用户最近浏览分类下的商品。

论文中可描述为：

“系统采用基于用户行为偏好的规则推荐方法，根据商品分类、热度和发布时间等因素生成推荐结果。”

## 五、实施与论文建议

为了更适合毕业设计推进，建议按以下顺序开发。

### 5.1 开发优先级建议

#### 第一阶段：基础功能

1. 普通用户注册、审核、登录。
2. 分类管理。
3. 商品发布、编辑、删除。
4. 商品列表、商品详情、搜索。

#### 第二阶段：完善业务功能

1. 求购信息功能。
2. 评论与回复功能。
3. 公告展示功能。
4. 个人中心功能。

#### 第三阶段：增强功能

1. 订单交易功能。
2. 推荐功能。
3. 收藏功能。
4. 邮件通知优化。

### 5.2 论文撰写建议

本项目适合按以下章节结构撰写论文：

1. 绪论
2. 相关技术介绍
3. 系统需求分析
4. 系统总体设计
5. 数据库设计
6. 系统详细设计与实现
7. 系统测试
8. 总结与展望

其中：

1. 第 3 章可重点写功能需求、角色需求和业务流程。
2. 第 4 章可重点写系统架构、模块划分和功能结构图。
3. 第 5 章可重点写 E-R 图和数据表设计。
4. 第 6 章可重点写注册审核、商品发布、订单交易、评论与推荐等核心功能实现。

### 5.3 总结

本系统采用校园实名审核与轻量交易流程，能够满足校园二手物品交易的基本需求。系统功能较为完整，技术路线清晰，业务难度适中，具有较强的课程设计与毕业设计适用性。
