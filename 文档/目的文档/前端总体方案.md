# 前端总体方案

## 一、文档归档规则

为保证前端文档分类清晰，约定如下：

1. 前端设计、页面方案、开发计划放在 `文档/目的文档`
2. 前端阶段进度、待办、开发顺序放在 `文档/进度文档`
3. 每完成一个前端页面模块后，必须同步一次进度文档
4. 每完成一个前端阶段后，必须补充阶段性记录

## 二、前端目标

本系统前端第一阶段以“演示优先”为目标，优先建设管理员后台页面，用于快速形成可截图、可演示、可联调的系统闭环。

前端第一阶段的核心目标：

1. 把已完成的后端能力接入后台管理端
2. 让管理员可以完成注册审核、分类维护、商品管理、求购管理、评论管理和订单管理
3. 为后续学生端页面开发和论文展示打基础

## 三、技术依赖与基座选择

后台前端基于现有 [yudao-ui-admin-vue3](D:/download/yudao-boot-mini-master-jdk17/yudao-ui-admin-vue3) 实现，不单独新建后台项目。

选择原因：

1. 该项目已有成熟的后台布局、路由、权限控制、表格和弹窗表单模式
2. 与当前后端接口风格一致，联调成本低
3. 适合毕设阶段快速完成“管理端可用”的结果

默认前端策略：

1. 首批只做管理员后台
2. 页面风格沿用现有后台
3. 不做明显重设计
4. 优先复用现有列表页、弹窗表单和详情弹窗模式

## 四、模块边界

前端第一阶段只覆盖后台管理端，不展开学生端页面的实现细节。

后台首批覆盖模块：

1. 用户审核与管理
2. 商品分类管理
3. 二手商品管理
4. 求购信息管理
5. 评论管理
6. 订单管理

不纳入第一阶段主方案的内容：

1. 学生端首页和商品浏览页
2. 收藏页单独实现
3. 图片管理页单独成页
4. 统计大屏和复杂报表

## 五、目录组织建议

建议在 `yudao-ui-admin-vue3` 中按业务模块新增目录：

1. `src/views/campus/user`
2. `src/views/campus/category`
3. `src/views/campus/goods`
4. `src/views/campus/wanted`
5. `src/views/campus/comment`
6. `src/views/campus/order`

接口层对应目录建议：

1. `src/api/campus/user`
2. `src/api/campus/category`
3. `src/api/campus/goods`
4. `src/api/campus/wanted`
5. `src/api/campus/comment`
6. `src/api/campus/order`

## 六、页面形态约定

后台页面统一使用现有管理端模式：

1. 顶部搜索区
2. 中部表格区
3. 底部分页区
4. 新增和修改使用弹窗表单
5. 详情使用只读弹窗或详情区域
6. 状态流转使用按钮、开关或下拉操作

## 七、接口对接原则

页面优先接当前已完成的后端接口，不额外扩大接口范围。

对接原则：

1. 每个页面先接分页和详情
2. 再接新增、修改、删除或状态操作
3. 复杂页面优先保证“可查、可改、可演示”
4. 不在第一阶段追求前端功能铺得过满

## 八、实施顺序概述

前端第一阶段建议按以下顺序推进：

1. 后台菜单与权限配置
2. 用户审核与分类页面
3. 商品与求购页面
4. 评论与订单页面
5. 前后端联调与演示优化

## 九、阶段目标

当前前端阶段的目标不是做一套全新的校园交易平台 UI，而是尽快把现有后端能力接成一套完整、稳定、适合答辩展示的管理端。
