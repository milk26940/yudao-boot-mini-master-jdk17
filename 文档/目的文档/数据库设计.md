# 校园二手物品交易管理系统数据库设计

## 文档定位

本文档用于说明校园二手物品交易管理系统的数据库设计方案，主要包括数据库设计原则、核心实体关系、主要数据表结构设计和关键字段说明。

本文档可作为毕业设计中的“数据库设计”章节基础材料。

---

## 一、数据库设计目标

本系统数据库设计主要围绕校园二手交易业务展开，目标如下：

1. 支持普通用户注册、审核、登录和资料维护。
2. 支持二手物品发布、修改、删除、浏览和搜索。
3. 支持求购信息发布与管理。
4. 支持商品评论与回复。
5. 支持简化交易订单流程。
6. 支持公告展示与后台管理。
7. 为后续功能扩展预留空间。

---

## 二、数据库设计原则

### 2.1 满足业务需求

数据库表结构应完整覆盖系统核心业务，包括用户、商品、求购、评论、订单和公告等主要模块。

### 2.2 结构清晰

每张表只负责一类核心业务数据，避免字段职责混乱，便于后续开发与维护。

### 2.3 易于扩展

对推荐、收藏、举报、聊天等可选功能预留扩展能力，避免后期大规模调整表结构。

### 2.4 便于论文展示

表数量控制在合理范围内，实体关系清晰，适合绘制 E-R 图和表结构说明表。

---

## 三、数据库总体设计

### 3.1 数据库名称建议

建议数据库名称：

`campus_second_trade`

### 3.2 核心实体

系统核心实体建议包括：

1. 用户
2. 商品分类
3. 二手商品
4. 商品图片
5. 求购信息
6. 评论
7. 订单
8. 订单日志
9. 收藏

### 3.3 实体关系说明

主要关系如下：

1. 一个用户可以发布多个商品。
2. 一个商品属于一个分类。
3. 一个商品可以对应多张图片。
4. 一个用户可以发布多条求购信息。
5. 一个商品可以对应多条评论。
6. 一条评论可以有上级评论，用于实现回复关系。
7. 一个商品最多对应一个有效订单流程。
8. 一个订单对应多个订单状态日志。
9. 一个用户可以收藏多个商品。

## 四、数据表设计

## 4.1 用户表 `campus_user`

### 4.1.1 表说明

用于保存普通学生用户信息，包括注册信息、审核信息、联系方式和账号状态。

### 4.1.2 字段设计

| 字段名 | 类型 | 主键 | 非空 | 说明 |
|---|---|---|---|---|
| id | bigint | 是 | 是 | 用户主键 |
| student_no | varchar(30) | 否 | 是 | 学号 |
| password | varchar(100) | 否 | 是 | 登录密码 |
| name | varchar(50) | 否 | 是 | 姓名 |
| gender | tinyint | 否 | 否 | 性别，1 男，2 女 |
| email | varchar(100) | 否 | 是 | 邮箱 |
| mobile | varchar(20) | 否 | 否 | 手机号 |
| qq | varchar(20) | 否 | 否 | QQ 号 |
| wechat | varchar(50) | 否 | 否 | 微信号 |
| avatar | varchar(255) | 否 | 否 | 头像地址 |
| student_card_url | varchar(255) | 否 | 是 | 学生证图片地址 |
| audit_status | tinyint | 否 | 是 | 审核状态 |
| audit_remark | varchar(255) | 否 | 否 | 审核备注 |
| status | tinyint | 否 | 是 | 账号状态 |
| login_time | datetime | 否 | 否 | 最后登录时间 |
| create_time | datetime | 否 | 是 | 创建时间 |
| update_time | datetime | 否 | 否 | 更新时间 |

### 4.1.3 字段说明

1. `student_no` 作为用户登录账号，符合校园场景。
2. `student_card_url` 用于保存学生证图片。
3. `audit_status` 用于控制注册审核流程。
4. `status` 用于控制账号是否可用。

### 4.1.4 审核状态建议

1. `0`：待审核
2. `1`：审核通过
3. `2`：审核驳回

### 4.1.5 账号状态建议

1. `0`：禁用
2. `1`：启用

## 五、索引设计建议

为了提升系统查询效率，建议建立以下索引：

1. `campus_user.student_no` 唯一索引
2. `campus_user.email` 普通索引
3. `second_goods.category_id` 普通索引
4. `second_goods.seller_id` 普通索引
5. `second_goods.status` 普通索引
