# 后台前端页面设计

## 一、页面范围

后台前端首批按“演示优先”原则，规划 6 个核心页面：

1. 用户审核与管理页
2. 商品分类管理页
3. 二手商品管理页
4. 求购信息管理页
5. 评论管理页
6. 订单管理页

## 二、用户审核与管理页

### 1. 页面目标

用于管理员审核学生注册申请、查看用户信息和维护用户状态。

### 2. 菜单位置

校园二手交易 / 用户审核与管理

### 3. 搜索区字段

1. 姓名
2. 学号
3. 邮箱
4. 审核状态
5. 账号状态
6. 注册时间

### 4. 表格字段

1. 用户编号
2. 姓名
3. 学号
4. 邮箱
5. 手机号
6. 审核状态
7. 账号状态
8. 注册时间

### 5. 主要操作

1. 查看详情
2. 审核通过
3. 审核驳回
4. 启用或禁用

### 6. 弹窗和详情结构

详情弹窗展示：

1. 基础信息
2. 联系方式
3. 学生证图片地址或预览位
4. 审核备注

### 7. 对应后端接口

1. 用户分页
2. 用户详情
3. 用户状态更新
4. 用户审核

### 8. 演示重点

突出“校园认证”和“管理员审核”这两个校园场景特色能力。

## 三、商品分类管理页

### 1. 页面目标

用于维护商品分类，支撑商品发布、筛选和统计展示。

### 2. 菜单位置

校园二手交易 / 商品分类管理

### 3. 搜索区字段

1. 分类名称
2. 状态

### 4. 表格字段

1. 分类编号
2. 分类名称
3. 排序
4. 状态
5. 创建时间

### 5. 主要操作

1. 新增
2. 修改
3. 删除
4. 启用或停用

### 6. 弹窗和详情结构

表单弹窗字段：

1. 分类名称
2. 排序
3. 状态

### 7. 对应后端接口

1. 分类分页
2. 分类详情
3. 分类新增
4. 分类修改
5. 分类删除
6. 分类简列表

### 8. 演示重点

适合体现后台基础数据维护能力，页面应保持简洁清晰。

### 1. 页面目标

用于统一查看和管理系统中的二手商品信息。

### 2. 菜单位置

校园二手交易 / 二手商品管理

### 3. 搜索区字段

1. 商品标题
2. 商品分类
3. 卖家编号
4. 商品状态
5. 价格区间
6. 发布时间

### 4. 表格字段

1. 商品编号
2. 商品标题
3. 分类名称
4. 卖家编号
5. 价格
6. 新旧程度
7. 商品状态
8. 发布时间

### 5. 主要操作

1. 查看详情
2. 修改
3. 删除
4. 调整状态

### 6. 弹窗和详情结构

详情弹窗展示：

1. 商品基础信息
2. 商品描述
3. 联系方式
4. 价格信息
5. 图片列表区域

### 7. 对应后端接口

1. 商品分页
2. 商品详情
3. 商品新增
4. 商品修改
5. 商品删除
6. 按商品编号查询图片列表

### 8. 演示重点

这是后台业务主页面，应作为演示重点页面之一。

## 五、求购信息管理页

### 1. 页面目标

用于管理用户发布的求购信息，形成“出售 + 求购”的双向业务场景。

### 2. 菜单位置

校园二手交易 / 求购信息管理

### 3. 搜索区字段

1. 标题
2. 发布用户
3. 状态
4. 预期价格区间
5. 创建时间

### 4. 表格字段

1. 编号
2. 标题
3. 用户编号
4. 品牌
5. 型号
6. 预期价格
7. 状态
8. 创建时间

### 5. 主要操作

1. 查看详情
2. 修改
3. 删除
4. 结束求购

### 6. 弹窗和详情结构

详情弹窗展示：

1. 标题
2. 品牌和型号
3. 预期价格
4. 描述
5. 联系方式

### 7. 对应后端接口

1. 求购分页
2. 求购详情
3. 求购新增
4. 求购修改
5. 求购删除

### 8. 演示重点

适合和商品页一起展示系统业务完整性。

5. 订单状态
6. 创建时间

### 4. 表格字段

1. 订单编号
2. 商品编号
3. 买家编号
4. 卖家编号
5. 订单金额
6. 配送方式
7. 订单状态
8. 创建时间

### 5. 主要操作

1. 查看详情
2. 修改
3. 删除
4. 状态更新

### 6. 弹窗和详情结构

详情弹窗展示：

1. 订单基础信息
2. 地址和备注

### 7. 对应后端接口

1. 订单分页
2. 订单详情
